## 📚 Descripción del Curso

En esta asignatura, aplicamos los principios fundamentales del paradigma de **Programación Orientada a Objetos (POO)** para construir software modular, escalable y fácil de mantener. El enfoque principal se divide en:

### 🛠️ Modelado de Clases y Objetos
Aprendemos a abstraer entidades del mundo real y convertirlas en **Clases** (plantillas) para generar **Objetos** (instancias) que interactúan entre sí dentro de un entorno controlado.

### 🏛️ Los Cuatro Pilares de la POO
Durante el desarrollo de las prácticas, implementamos los conceptos clave que sostienen este paradigma:

1.  **Abstracción:** Identificación de las características esenciales de un objeto, eliminando detalles innecesarios para simplificar el diseño.
2.  **Encapsulamiento:** Protección de los datos internos mediante el uso de modificadores de acceso y métodos *getter/setter*.
3.  **Herencia:** Creación de nuevas clases basadas en clases existentes para fomentar la reutilización de código y jerarquizar conceptos.
4.  **Polimorfismo:** Capacidad de que un mismo método o mensaje se comporte de distintas formas según el objeto que lo invoque.

## 📐 Buenas Prácticas y Diseño

Más allá de la escritura de código, el curso enfatiza la planificación estructural:
* **Diagramas de Clase (UML):** Diseño previo de la arquitectura del sistema antes de la implementación.
* **Modularidad:** División del sistema en partes independientes para facilitar el mantenimiento.
* **Documentación:** Uso de comentarios claros y estructuras de archivos organizadas.
