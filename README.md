El enfoque principal se divide en:🛠️ Modelado de Clases y ObjetosAprendemos a abstraer entidades del mundo real y convertirlas en Clases (plantillas) para generar Objetos (instancias) que interactúan entre sí dentro de un entorno controlado.🏛️ Los Cuatro Pilares de la POODurante el desarrollo de las prácticas, implementamos los conceptos clave que sostienen este paradigma:Abstracción: Identificación de las características esenciales de un objeto, eliminando detalles innecesarios para simplificar el diseño.Encapsulamiento: Protección de los datos internos mediante el uso de modificadores de acceso y métodos getter/setter.Herencia: Creación de nuevas clases basadas en clases existentes para fomentar la reutilización de código y jerarquizar conceptos.Polimorfismo: Capacidad de que un mismo método o mensaje se comporte de distintas formas según el objeto que lo invoque.📐 Buenas Prácticas y DiseñoMás allá de la escritura de código, el curso enfatiza la planificación estructural:Diagramas de Clase (UML): Diseño previo de la arquitectura del sistema antes de la implementación.Modularidad: División del sistema en partes independientes para facilitar el mantenimiento.Documentación: Uso de comentarios claros y estructuras de archivos organizadas.📂 Estructura del Repositorio/Practicas: Ejercicios semanales realizados en clase./Proyectos: Aplicaciones integradoras que combinan todos los pilares de la POO./Documentacion: Diagramas UML y reportes de prácticas.Este repositorio es de carácter académico y sirve como evidencia de aprendizaje para el semestre actual.
